Hexo基础

与普通文本文档一样，博客网页的写作，通常包括内容创作和样式渲染两个步骤。从功能上来看，Hexo是一个与Latex类似的工具。在Hexo框架下，内容创作和样式渲染被分隔开来。作者只需专注于内容创作，结合样式渲染规则，即可快速生成完整的博客网页。在内容创作时，Hexo采用了Markdown标记语言来对内容元素所对应的格式进行标记；在样式渲染时，Hexo根据Markdown的标记和主题配置文件来生成HTML网页。

Hexo环境配置，可参考官方文档，其指导非常详细。

样式渲染

Hexo的主题样式渲染规则文件是一种 Embedded JavaScript（.ejs）文件，主要涉及到layout.ejs和<theme_layout>.ejs两个文件。layout.ejs是网页整体框架的渲染规则，对于网页的主要内容，其渲染规则为<theme_layout>.ejs规。对于每一份稿件，其抬头的layout键参数，便是用于指定<theme_layout>.ejs。例如，若稿件抬头定义layout: selflayout，那么Hexo就会使用selflayout.ejs规则对该稿件进行渲染。如果稿件抬头没有指定layout，或者所指定的layout规则不存在，Hexo都会使用默认layout值，这个默认值定义在站点配置文件_config.yml中，通常就是post。Hexo内置的渲染规则文件放置在了node_modules/hexo-theme-landscape/layout/文件夹下面。

稿件正文

Hexo稿件正文的撰写需遵守Markdown语法，这里不再赘述。只是对常用的几个额外功能进行记录。

本地图片引用

  • 绝对路径引用：

    • 当Hexo项目中只用到少量图片时，可以将图片统一放在sources/images文件夹中
    • 在Markdown文件中用![description](images/name.format)的方式来引用
    • 图片既可以在首页内容中访问到，也可以在文章正文中访问到。

  • 相对路径引用：

    • 图片除了可以放在统一的images文件夹中，还可以放在稿件附属文件夹中
    • 在站点配置文件_config.yml中，设置post_asset_folder: true键值对后，在使用hexo new命令新建稿件时，同时会创建同名附属文件，用于存放图片
    • 在Markdown中使用![](name.format)来进行引用
    • 这种相对路径的图片引用方法在博文详情页面显示没有问题，但是在首页预览页面图片将显示不出来

  • 标签插件引用：

    • 希望图片在文章和首页中同时显示，可以使用标签插件引用
    • 这也是一种相对路径引用，将图片存放在稿件附属文件夹中
    • 在Markdown文件中使用{% asset_img example.jpg This is an example image %}格式来引用图片
